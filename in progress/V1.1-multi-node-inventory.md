# V1.1 Multi-Node — Work Inventory

## Prerequisites
- Node 1: Current machine (bootstrap node, V1 running)
- Node 2: Fresh Linux Mint install
- Node 3: Fresh Linux Mint install
- All on same LAN with static IPs or DHCP reservations
- SSH access between nodes
- Firewall open: 2379-2380 (etcd), 10000-20000 (services), 8443 (gateway)

---

## 1. Cluster Join Flow

- [ ] `globular cluster token create` on Node 1 (generates join token)
- [ ] `globular cluster join --token <token> --controller <node1-ip>:12000` on Node 2 & 3
- [ ] Node agent registers with controller, gets TLS certs issued
- [ ] Verify all 3 nodes visible in `globular cluster list-nodes`

---

## 2. etcd Cluster (Single → 3-Node)

Current: single-node etcd on Node 1.

- [ ] Expand etcd from 1-node to 3-node cluster
- [ ] Each node runs its own etcd member
- [ ] etcd config must include all member addresses (peer URLs)
- [ ] `initial-cluster` parameter needs all 3 nodes
- [ ] Controller must update etcd endpoints in all services when members change
- [ ] Verify quorum: kill one node, cluster still works
- [ ] etcd TLS: peer certificates for inter-node communication

Pattern: etcd.conf.yaml has `initial-cluster: node1=https://ip1:2380,node2=https://ip2:2380,node3=https://ip3:2380`

---

## 3. MinIO Distributed Mode

Current: single-node MinIO on Node 1.

- [ ] Expand to distributed MinIO (erasure coding across nodes)
- [ ] Minimum: 4 drives across nodes (or 1 drive per node × 3 nodes with replication)
- [ ] MinIO config needs all node endpoints: `https://node1/data https://node2/data https://node3/data`
- [ ] Update MINIO_VOLUMES environment variable
- [ ] TLS between MinIO nodes
- [ ] Verify: upload file, kill one node, file still accessible

Pattern: MinIO startup args list all node endpoints.

---

## 4. ScyllaDB Cluster

Current: single-node ScyllaDB on Node 1.

- [ ] Add Node 2 & 3 as ScyllaDB seeds
- [ ] `scylla.yaml` needs `seed_provider` with all node IPs
- [ ] New nodes auto-join via gossip protocol (Scylla handles this)
- [ ] Verify: `nodetool status` shows 3 nodes UN (Up Normal)
- [ ] Replication factor: update keyspaces to RF=3
- [ ] TLS: inter-node encryption

Pattern: `scylla.yaml` → `seeds: "ip1,ip2,ip3"` + `ALTER KEYSPACE ... WITH replication = {'class': 'NetworkTopologyStrategy', 'datacenter1': 3}`

---

## 5. Common Pattern: Node Address Sync

All three (etcd, MinIO, ScyllaDB) follow the same pattern:
- Config file has a list of node addresses
- When a node joins/leaves, the list must be updated on ALL nodes
- The controller should orchestrate this via desired state + reconciliation

### Proposed approach:
- [ ] Controller tracks cluster membership (already does via node-agent heartbeat)
- [ ] When membership changes, controller generates new config for etcd/minio/scylla
- [ ] Config pushed as desired state → node agent applies
- [ ] Rolling restart: update one node at a time (etcd quorum safety)

---

## 6. Keepalived (Virtual IP)

For a single entry point to the cluster (floating VIP):

- [ ] Install keepalived on all 3 nodes
- [ ] Configure VRRP: one master, two backups
- [ ] VIP floats to healthy node if master fails
- [ ] Health check script: check if gateway is responding
- [ ] DNS points to VIP (not individual node IPs)
- [ ] Gateway/Envoy on each node handles requests

Pattern: `/etc/keepalived/keepalived.conf` with `vrrp_instance`, priority per node, `track_script` for health.

---

## 7. Service Distribution

- [ ] Controller decides which services run where (scheduling)
- [ ] Core services (etcd, controller, auth, rbac, resource, event) on all 3 nodes
- [ ] Application services can be distributed or replicated
- [ ] AI stack: watcher on all nodes, executor on 1-2 nodes
- [ ] Envoy/gateway on all nodes (behind keepalived VIP)

## 9. TLS for Multi-Node

- [ ] CA shared across cluster (already done — CA is on bootstrap node)
- [ ] Each joining node gets its own service certificate issued
- [ ] Peer certificates for etcd inter-node communication
- [ ] Client certificates for cross-node service-to-service calls
- [ ] Certificate distribution via join flow

---

## 10. Failure Testing (Definition of Done)

- [ ] Kill Node 2 → etcd quorum holds, services redistribute
- [ ] Kill Node 3 → same
- [ ] Kill Node 1 (bootstrap) → VIP floats, cluster still serves
- [ ] Bring killed node back → auto-rejoins, state syncs
- [ ] Network partition (iptables) → split-brain handling
- [ ] ScyllaDB node down → reads/writes still work (RF=3)
- [ ] MinIO node down → files still accessible
