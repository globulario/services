ClusterDoctorService — Implementation Plan (revised)
======================================================

Overview
--------
New read-only Globular microservice providing deterministic operational intelligence. It sits on top
of ClusterControllerService and NodeAgentService, never mutates state, and returns structured findings
with full evidence trails.

Incorporates adjustments from adjustment.txt (2026-02-23).
Revised to match clustercontroller / nodeagent conventions (shared proto dir, client package,
Envoy/xDS registration, TS reachability).

3. Proto Definition (services/proto/clusterdoctor.proto)
--------------------------------------------------------
  File location : services/proto/clusterdoctor.proto
  Generated Go  : services/golang/clusterdoctor/clusterdoctorpb/

  Protoc invocation (mirrors other services in the repo):
    protoc -I services/proto \
      --go_out=services/golang/clusterdoctor/clusterdoctorpb \
      --go-grpc_out=services/golang/clusterdoctor/clusterdoctorpb \
      services/proto/clusterdoctor.proto

  syntax = "proto3";
  package clusterdoctor;
  option go_package = "github.com/globulario/services/golang/clusterdoctor/clusterdoctorpb;clusterdoctorpb";
  import "google/protobuf/timestamp.proto";

  enum Severity {
    SEVERITY_UNKNOWN  = 0;
    SEVERITY_INFO     = 1;
    SEVERITY_WARN     = 2;
    SEVERITY_ERROR    = 3;
    SEVERITY_CRITICAL = 4;
  }

  enum ClusterStatus {
    CLUSTER_STATUS_UNKNOWN = 0;
    CLUSTER_HEALTHY        = 1;
    CLUSTER_DEGRADED       = 2;
    CLUSTER_CRITICAL       = 3;
  }

  enum DriftCategory {
    DRIFT_UNKNOWN           = 0;
    MISSING_UNIT_FILE       = 1;
    UNIT_STOPPED            = 2;
    UNIT_DISABLED           = 3;
    VERSION_MISMATCH        = 4;
    STATE_HASH_MISMATCH     = 5;  // ← renamed from CONFIG_HASH_MISMATCH; covers services + network hash
    ENDPOINT_MISSING        = 6;
    INVENTORY_INCOMPLETE    = 7;
  }

  enum PlanRisk {
    PLAN_RISK_UNKNOWN   = 0;
    PLAN_RISK_SAFE      = 1;
    PLAN_RISK_MODERATE  = 2;
    PLAN_RISK_DANGEROUS = 3;
  }

  enum InvariantStatus {
    INVARIANT_UNKNOWN = 0;
    INVARIANT_PASS    = 1;
    INVARIANT_FAIL    = 2;
    INVARIANT_PENDING = 3;  // upstream RPC / data not yet available
  }

  message ReportHeader {
    google.protobuf.Timestamp generated_at = 1;
    string snapshot_id                     = 2;
    string globular_version                = 3;
    repeated string data_sources           = 4;  // e.g. "clustercontroller.ListNodes"
    bool data_incomplete                   = 5;
    repeated Evidence data_errors          = 6;
  }

  message Evidence {
    string source_service               = 1;
    string source_rpc                   = 2;
    map<string, string> key_values      = 3;
    google.protobuf.Timestamp timestamp = 4;
  }

  message RemediationStep {
    uint32 order            = 1;
    string description      = 2;
    string cli_command      = 3;  // optional: globularcli command
    string future_action_id = 4;  // optional: for v2 approved execution
  }

  message Finding {
    string finding_id                    = 1;  // deterministic: sha256(invariant_id+":"+entity_ref+":"+primary_evidence_key)[:16]
    string invariant_id                  = 2;
    Severity severity                    = 3;
    string category                      = 4;
    string entity_ref                    = 5;  // node_id / unit_name / service
    string summary                       = 6;
    repeated Evidence evidence           = 7;
    repeated RemediationStep remediation = 8;
    InvariantStatus invariant_status     = 9;
  }

  // --- Cluster Report ---

  message ClusterReportRequest {}

  message ClusterReport {
    ReportHeader header                    = 1;
    ClusterStatus overall_status           = 2;
    repeated Finding findings              = 3;
    map<string, uint32> counts_by_category = 4;
    repeated string top_issue_ids          = 5;
  }

  // --- Node Report ---

  message NodeReportRequest { string node_id = 1; }

  message NodeReport {
    ReportHeader header             = 1;
    string node_id                  = 2;
    bool reachable                  = 3;
    int64 heartbeat_age_seconds     = 4;
    repeated Finding findings       = 5;
  }

  // --- Drift Report ---

  message DriftReportRequest { string node_id = 1; }  // empty = all nodes

  message DriftItem {
    string node_id             = 1;
    string entity_ref          = 2;
    DriftCategory category     = 3;
    string desired             = 4;
    string actual              = 5;
    repeated Evidence evidence = 6;
  }

  message DriftReport {
    ReportHeader header        = 1;
    repeated DriftItem items   = 2;
    uint32 total_drift_count   = 3;
  }

  // --- Explain Finding ---

  message ExplainFindingRequest { string finding_id = 1; }

  message FindingExplanation {
    string finding_id                    = 1;
    string invariant_id                  = 2;
    string why_failed                    = 3;
    repeated RemediationStep remediation = 4;
    repeated Evidence evidence           = 5;
    PlanRisk plan_risk                   = 6;
    repeated string plan_diff            = 7;
  }

  service ClusterDoctorService {
    rpc GetClusterReport(ClusterReportRequest) returns (ClusterReport);
    rpc GetNodeReport(NodeReportRequest)       returns (NodeReport);
    rpc GetDriftReport(DriftReportRequest)     returns (DriftReport);
    rpc ExplainFinding(ExplainFindingRequest)  returns (FindingExplanation);
  }

4. Configuration (clusterdoctor_config.json)
--------------------------------------------
  Default config path: /var/lib/globular/clusterdoctor/config.json
  Passed via CLI flag: --config (mirrors clustercontroller pattern)

  Port MUST be set in config — server refuses to start without it.
  Check the existing port allocation table in the repo before assigning
  (clustercontroller = 12000, nodeagent = 11000).

  {
    "port": 0,                           // required; must be non-zero
    "controller_endpoint": "",           // required; address of ClusterControllerService
    "snapshot_ttl_seconds": 5,           // SnapshotCache TTL
    "node_heartbeat_stale_seconds": 120, // threshold for node.reachable invariant
    "upstream_list_timeout_seconds": 10, // timeout for ClusterController list RPCs
    "upstream_node_timeout_seconds": 5,  // timeout for per-node NodeAgent RPCs
    "upstream_node_concurrency": 20,     // max parallel per-node goroutines
    "emit_audit_events": false           // default off; set true to write to EventService
  }

  Load via config.go at startup. All rule thresholds are passed as parameters to invariants
  (no hardcoded constants in rule files).

8. Collector Layer (collector/collector.go)
-------------------------------------------
  Gathers data from upstream concurrently. Uses separate deadlines for list vs per-node calls
  (from config). Per-node goroutines are capped at cfg.UpstreamNodeConcurrency (default 20)
  using a semaphore channel to avoid unbounded fan-out.

  Partial failure → DataIncomplete = true + error evidence stored in DataErrors.
  Never panics or silently swallows errors.

  RPCs called (read-only):
  ─────────────────────────────────────────────────────────────────────────────
  Call              : ClusterController.ListNodes
  Request           : clustercontrollerpb.ListNodesRequest{}
  Returns           : []*NodeRecord (NodeId, LastSeen, Status, AgentEndpoint, Identity)
  Timeout           : cfg.UpstreamListTimeout
  ─────────────────────────────────────────────────────────────────────────────
  Call              : ClusterController.GetClusterHealthV1
  Request           : GetClusterHealthV1Request{}
  Returns           : []*NodeHealth (DesiredServicesHash, AppliedServicesHash,
                      DesiredNetworkHash, AppliedNetworkHash, CurrentPlanId, LastError)
  Timeout           : cfg.UpstreamListTimeout
  ─────────────────────────────────────────────────────────────────────────────
  Call              : ClusterController.GetNodePlanV1  (per-node)
  Returns           : *planpb.NodePlan (Spec.Steps, Spec.Desired.Services[].Version,
                      Generation, DesiredHash)
  Timeout           : cfg.UpstreamNodeTimeout
  ─────────────────────────────────────────────────────────────────────────────
  Call              : NodeAgent.GetInventory  (per-node)
  Returns           : *Inventory (Identity, UnixTime, InventoryComplete bool,
                      Components[]{Name,Version,Installed}, Units[]{Name,State})
  Timeout           : cfg.UpstreamNodeTimeout
  ─────────────────────────────────────────────────────────────────────────────
  Call              : NodeAgent.GetPlanStatusV1  (per-node)
  Returns           : *planpb.NodePlanStatus (State, Steps[]{Id,State,Message},
                      ErrorMessage, ErrorStepId)
  Timeout           : cfg.UpstreamNodeTimeout
  ─────────────────────────────────────────────────────────────────────────────

  NodeAgent connections are dialed per-node using NodeRecord.AgentEndpoint from ListNodes.
  Connection pool is rebuilt whenever the node list changes.

  Snapshot struct:

    type Snapshot struct {
        SnapshotID     string
        GeneratedAt    time.Time
        DataSources    []string
        DataIncomplete bool
        DataErrors     []DataError

        Nodes        []*clustercontrollerpb.NodeRecord
        NodeHealths  map[string]*clustercontrollerpb.NodeHealth  // keyed by NodeId
        Inventories  map[string]*nodeagentpb.Inventory           // keyed by NodeId
        PlanStatuses map[string]*planpb.NodePlanStatus           // keyed by NodeId
        NodePlans    map[string]*planpb.NodePlan                 // keyed by NodeId
    }

  SnapshotCache:
  - Mutex-protected, TTL from cfg.SnapshotTTL
  - Singleflight via golang.org/x/sync/singleflight — in-flight fetches are shared,
    no stampede on concurrent UI requests
  - Cache keyed by request scope (cluster-wide vs single node_id)

9. Unit State Normalization (rules/unitstate.go)
------------------------------------------------
  All invariants use UnitStateEnum constants — never raw strings.
  Defined once here; avoids brittle string comparisons scattered across rule files.

    type UnitStateEnum int

    const (
        UnitStateUnknown  UnitStateEnum = iota
        UnitStateActive
        UnitStateInactive
        UnitStateFailed
        UnitStateDisabled
        UnitStateNotFound
        UnitStateActivating
        UnitStateDeactivating
    )

    // NormalizeUnitState maps agent-reported strings (systemd output, agent mapping)
    // to a canonical enum. Add new strings here as the agent evolves.
    func NormalizeUnitState(s string) UnitStateEnum {
        switch strings.ToLower(strings.TrimSpace(s)) {
        case "active", "running":
            return UnitStateActive
        case "inactive", "dead":
            return UnitStateInactive
        case "failed":
            return UnitStateFailed
        case "disabled":
            return UnitStateDisabled
        case "not-found", "not_found", "notfound", "missing":
            return UnitStateNotFound
        case "activating":
            return UnitStateActivating
        case "deactivating":
            return UnitStateDeactivating
        default:
            return UnitStateUnknown
        }
    }

10. Finding ID Generation (rules/invariant.go)
---------------------------------------------
  finding_id must be deterministic so that ExplainFinding(finding_id) works reliably
  across repeated calls with unchanged cluster state.

  Algorithm:
    raw    = invariant_id + ":" + entity_ref + ":" + primary_evidence_key
    digest = sha256(raw)
    id     = hex(digest[:8])   // 16 hex chars, stable

  primary_evidence_key is the most discriminating field for that invariant
  (e.g. node_id for node-scoped, unit_name for unit-scoped).

  finding_id does NOT include snapshot_id, so the same condition produces the same
  ID across snapshots (stable for UI linking, drilldown, deduplication).

11. Rules Layer — Invariant Map
--------------------------------

  11.1 Invariant interface

    type Invariant interface {
        ID()       string  // stable, e.g. "node.reachable"
        Category() string  // "availability"/"drift"/"systemd"/"plan"/etc.
        Scope()    string  // "cluster" | "node" | "service"
        Evaluate(snap *collector.Snapshot, cfg *Config) []Finding
    }

  11.2 Implementable in v1 (available RPC data)

  ─────────────────────────────────────────────────────────────────────────────
  invariant_id      : node.reachable
  Severity          : CRITICAL
  Source RPCs       : CC.ListNodes
  Failure condition : NodeRecord.LastSeen older than cfg.NodeHeartbeatStale
                      OR NodeRecord.Status == "unreachable"
  Evidence fields   : last_seen, age_sec, status
  Remediation       : check nodeagent service, inspect logs, verify network
  ─────────────────────────────────────────────────────────────────────────────
  invariant_id      : node.inventory.complete
  Severity          : ERROR
  Source RPCs       : NA.GetInventory
  Failure condition : Primary — Inventory.InventoryComplete == false (explicit flag)
                      Fallback (if flag absent) — Components empty OR Units empty
                      If neither signal available → INVARIANT_PENDING
  Evidence fields   : inventory_complete_flag, component_count, unit_count
  Remediation       : re-trigger inventory scan on node agent
  ─────────────────────────────────────────────────────────────────────────────
  invariant_id      : node.plan.last_apply_success
  Severity          : ERROR
  Source RPCs       : NA.GetPlanStatusV1
  Failure condition : NodePlanStatus.State == PLAN_FAILED
                      OR any StepStatus.State == STEP_FAILED
  Evidence fields   : plan_id, state, error_message, error_step_id, failed_step_ids
  Remediation       : inspect plan logs, retry or roll back plan
  ─────────────────────────────────────────────────────────────────────────────
  invariant_id      : node.systemd.unit_files_present
  Severity          : ERROR
  Source RPCs       : NA.GetInventory
  Failure condition : any unit where NormalizeUnitState(UnitStatus.State) == UnitStateNotFound
  Evidence fields   : unit_name, raw_state
  Remediation       : reinstall service package; check repository connectivity
  ─────────────────────────────────────────────────────────────────────────────
  invariant_id      : node.systemd.units_running
  Severity          : WARN
  Source RPCs       : NA.GetInventory
  Failure condition : any unit where NormalizeUnitState(UnitStatus.State) == UnitStateFailed
                      OR UnitStateInactive (for units expected to be active)
  Evidence fields   : unit_name, raw_state, normalized_state
  Remediation       : restart unit; check systemd journal
  ─────────────────────────────────────────────────────────────────────────────
  invariant_id      : cluster.services.drift
  Severity          : WARN
  Source RPCs       : CC.GetClusterHealthV1
  Failure condition : NodeHealth.DesiredServicesHash != NodeHealth.AppliedServicesHash
  Evidence fields   : node_id, desired_hash, applied_hash
  Remediation       : trigger reconciliation; inspect desired vs actual service state
  ─────────────────────────────────────────────────────────────────────────────
  invariant_id      : cluster.network.drift
  Severity          : WARN
  Source RPCs       : CC.GetClusterHealthV1
  Failure condition : NodeHealth.DesiredNetworkHash != NodeHealth.AppliedNetworkHash
  Evidence fields   : node_id, desired_hash, applied_hash
  Remediation       : trigger network reconciliation; check cluster network config
  ─────────────────────────────────────────────────────────────────────────────

  11.3 PENDING invariants (missing upstream RPCs — returned with INVARIANT_PENDING, never faked)

  ─────────────────────────────────────────────────────────────────────────────
  invariant_id      : cluster.repo.reachable
  Missing data      : No repository health RPC in ClusterController
  Proposed addition : Add GetRepositoryStatus() returns (RepositoryStatus) to
                      ClusterControllerService
  ─────────────────────────────────────────────────────────────────────────────
  invariant_id      : cluster.discovery.consistent
  Missing data      : No discovery consistency RPC
  Proposed addition : Add GetDiscoveryStatus() to ClusterControllerService or DiscoveryService
  ─────────────────────────────────────────────────────────────────────────────
  invariant_id      : security.certs.not_expired
  Missing data      : No certificate expiry RPC
  Proposed addition : Add GetCertificateStatus(node_id) to NodeAgentService or SecurityService
  ─────────────────────────────────────────────────────────────────────────────

12. DriftReport Computation
---------------------------
  Uses Snapshot.NodeHealths + Snapshot.Inventories + Snapshot.NodePlans.

  Source                  Field                                         → DriftCategory
  ─────────────────────────────────────────────────────────────────────────────────────
  NodeHealth              DesiredServicesHash != AppliedServicesHash    STATE_HASH_MISMATCH
  NodeHealth              DesiredNetworkHash != AppliedNetworkHash      STATE_HASH_MISMATCH
  UnitStatus              NormalizeUnitState == UnitStateNotFound       MISSING_UNIT_FILE
  UnitStatus              NormalizeUnitState == UnitStateInactive       UNIT_STOPPED
  UnitStatus              NormalizeUnitState == UnitStateDisabled       UNIT_DISABLED
  InstalledComponent      .Installed == false                           INVENTORY_INCOMPLETE
  InstalledComponent      .Version != NodePlan.Spec.Desired.Services[matching].Version
                          (when plan args contain version)              VERSION_MISMATCH

  DriftItem.desired and DriftItem.actual hold human-readable strings
  (hash values, version strings, state names) for UI display.

13. Security
------------
  - All RPCs require auth token (same globular_service interceptor chain)
  - RBAC: cluster.read / node.read roles required
  - Evidence must never include secret values (tokens, private keys, passwords)
  - Audit events to EventService: controlled by cfg.EmitAuditEvents (default false)
    When enabled: emit on each report request (who, what scope, timestamp) and on
    upstream RPC failures

  ├── cluster_services_drift_test.go    — matching hashes → PASS; mismatch → FAIL

  1. Construct a collector.Snapshot with mock clustercontrollerpb / nodeagentpb / planpb structs
  2. Call invariant.Evaluate(snap, cfg) with a test Config
  3. Assert finding count, InvariantID, Severity, finding_id stability (same input → same id),

17. Implementation Sequence
----------------------------
  1. Proto + codegen
     Write services/proto/clusterdoctor.proto (section 3), run protoc, commit generated
     files into clusterdoctor/clusterdoctorpb/.

  2. Config
     Implement config.go: load/validate clusterdoctor_config.json; fail fast if port
     or controller_endpoint is missing (section 4).

  3. Collector + Snapshot
     Implement collector.go + snapshot.go with singleflight cache (section 8).
     Write an integration harness (main_debug.go build-tagged) that prints raw snapshot
     JSON against a live cluster for manual verification.

  4. Unit state normalization
     Implement unitstate.go + unitstate_test.go (section 9). All strings pass through
     NormalizeUnitState before any invariant evaluation.

  5. Rules layer
     Implement each invariant + unit tests (sections 11, 14). All tests must pass with
     mock data before any server code is written.

  6. Render layer
     Build ClusterReport / NodeReport / DriftReport protobuf responses from findings.

  7. Server skeleton
     main.go + server.go wiring collector → rules → render (section 5).
     Register service with config.SaveServiceConfiguration (Id = "clusterdoctor.ClusterDoctorService").

  8. Go client library
     Implement clusterdoctor_client/ (section 6) so other Go services can call the doctor.

  9. Pending invariant stubs
     Add pending.go returning INVARIANT_PENDING findings with proposed upstream RPC
     descriptions so the UI can surface them (section 11.3).

  10. TypeScript client + Admin UI
      Add TS client (section 7) and wire cluster_overview, cluster_nodes,
      cluster_reconciliation pages (section 15).

  11. System test
      Deploy to dev cluster. Verify:
      - GetClusterReport is idempotent (same result on repeated calls, same cluster state)
      - GetDriftReport categorises at least one drift condition correctly
