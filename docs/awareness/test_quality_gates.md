# Test Quality Gates

## The problem: scaffold stubs create false confidence

When `awareness enforce scaffold` generates required test stubs, it writes:

```go
func TestSomething(t *testing.T) {
    t.Skip("TODO: implement required awareness test")
}
```

These stubs make CI green — the test exists, it doesn't fail — but they provide no real verification. A DONE fix case verified only by scaffold stubs is falsely marked complete.

## CI gates

The `globular awareness enforce` command exposes two CI thresholds:

### `--max-required-test-no-path`

Limits the number of `REQUIRED_TEST_NO_PATH` findings (tests listed in fix cases but not resolvable to an actual file path).

```bash
# Fail CI if any required tests cannot be located
globular awareness enforce --max-required-test-no-path 0
```

Set to `-1` to disable (never fail on this count).

### `--max-todo-scaffold-skips`

Limits the number of test files that contain `t.Skip("TODO: implement required awareness test")`.

```bash
# Zero tolerance for scaffold stubs
globular awareness enforce --max-todo-scaffold-skips 0

# Allow up to 5 during active backlog work
globular awareness enforce --max-todo-scaffold-skips 5
```

Set to `-1` to disable (default).

## Scaffold detection

`enforce.ScanScaffoldTests(repoRoot, docsDir)` walks all `*_test.go` files looking for:

- `t.Skip("TODO: implement required awareness test")`
- `t.Skip("TODO")`
- Similar scaffold marker patterns

It also cross-references DONE fix cases in `fix_cases.yaml`. If a DONE fix case has all its required tests as scaffold stubs, it emits a `DONE_FIXCASE_SCAFFOLD_ONLY` ERROR finding — a DONE status verified only by stubs is a regression.

## Workflow

1. Scaffold stubs are generated by `globular awareness enforce scaffold`
2. Write the real test logic (replace `t.Skip("TODO: ...")`)
3. Run `globular awareness enforce --max-todo-scaffold-skips 0` to verify no stubs remain
4. Add the test name to `fix_cases.yaml` `tests` list
5. Rebuild and reload the awareness graph

## Tests

- `TestScaffoldScan_DetectsTodoSkips` — stub detection works
- `TestScaffoldScan_ExcludesVendor` — vendor paths are ignored
- `TestGraphIntegrity_DoneFixCaseWithTodoSkipFails` — DONE + scaffold → ERROR
- `TestAwarenessAudit_MaxTodoScaffoldSkipsZero` — CLI gate enforces zero threshold
- `TestCoverageReport_ScaffoldTestsCountedSeparately` — scaffold stubs are not counted as real test coverage
