# Task Curation

Curate from real repository history. Do not invent a task.

A candidate should include:

- external repository identity,
- historical base revision,
- oracle revision or sealed oracle pointer,
- changed paths,
- brief task summary,
- why it is architecture-relevant,
- risks of oracle leakage.

Stop for human selection before freezing. A shortlist is not a benchmark result,
and a selected task is not proof until the blind workflow completes.
